# UML包含和扩展的关系

## 包含

```
1、体现重用的包含关系。当两个或多个用例中共用一组相同的动作，这时可以将这组相同的动作抽出来作为一个独立的用例（被包含用例），供多个基础用例所共享。因为共同行为用例被抽出，基用例并非一个完整的用例，所以include关系中的基用例在执行时，被包含用例也必然被执行；
2、简化事件流的包含关系。有时当某用例的事件流过于复杂时，为了简化用例的描写叙述，可以抽象出一个基用例，来包含相关颗粒的用例。
```

## 扩展

```
用于分离出不同的行为。一个用例明显地混合了两种或者两种以上的不同场景，即依据情况可能发生多种分支，则能够将这个用例分为一个基本用例和一个或多个扩展用例。这样可能会使描写叙述更加清晰，扩展用例为基用例加入新的行为，扩展关系用《extend》表示，箭头指向“被扩展”的基础用例。基础用例执行时，扩展用例不一定执行。
```

# 计算机周期

## 时钟周期

```
时钟周期：
    是处理操作的最基本单位；
    通常称为 节拍脉冲或 T 周期；
```

## 机器周期

```
机器周期：
    也称为 CPU 周期；
    一条指令的执行过程划分为若干个阶段（如取指、译码、执行等），每一个阶段完成一个基本操作，完成一个基本操作所需要的时间称为机器周期；
    常用从内存中读取一个指令字的最短时间来规定 CPU 周期；
    一个机器周期由若干个状态周期组成；
```

## 指令周期

```
指令周期：
    执行一条指令所需要的时间，一般由若干个机器周期组成；
    指令不同，所需的机器周期数也不同，常常用若干个 CPU 周期数来表示；
    一条指令的取出阶段需要一个 CPU 周期时间，而一个 CPU 周期时间又包含若干个时钟周期，这些时钟周期的总和，决定了一个 CPU 周期的时间宽度；
    取出和执行任何一条指令所需的最短时间为两个 CPU 周期；
    对于复杂一些的指令，则需要更多的 CPU 周期；
```

## 存储周期

```
存储周期：
    指主存储器两次启动操作之间需要的最小时间间隔，也称之为主存储器周期时间，稍长于主存读写存取周期；
    CPU 通过使用AR（地址寄存器）、DR（数据缓冲寄存器）、总线与主存进行数据传送；
    连续完成两次读/写操作中间需待前一操作信号全部消失后才能进行下一个操作，故半导体存储器的周期时间通常稍大于其存取时间，而磁芯存储器的周期时间通常是其存取时间的两倍；
```

## 总线周期

```
总线周期：
    主存和I/O端口是挂接在总线上的，CPU对主存和I/O接口的访问是通过总线实现的；
    CPU 通过总线对微处理器外部的主存或I/O接口进行一次访问所需的时间成为一个总线周期；
    一个总线周期一般包含 4 个时钟周期，分别称为 4 个状态（T1状态、T2状态、T3状态、T4状态）；
```

# 大根堆

```
1、将数据按照顺序构建一颗完全二叉树；
2、从最后一个非终端节点开始构建，把三个值当中最大的放在根的位置；
3、找下一个非终端节点进行同样的操作；
4、直至找到整个二叉树的根节点；
5、再回来查询有没有由于上层的交换最大值的操作造成下层已经交换过的根的大小；

2,8,7,1,3,5,6,4；
第一步：构造完全二叉树
                  2
           |--------------|
           8              7
      |---------|    |---------|
      1         3    5         6
   |---
   4

第二步：首先找到最后一个非终端节点，1比4小，交换位置
                  2
           |--------------|
           8              7
      |---------|    |---------|
      4         3    5         6
   |---
   1
   
第三步：找到7节点，7都比5、6大，不交换位置
                  2
           |--------------|
           8              7
      |---------|    |---------|
      4         3    5         6
   |---
   1   

第四步：找到8节点，8都比4、3大，不交换位置
                  2
           |--------------|
           8              7
      |---------|    |---------|
      4         3    5         6
   |---
   1  
   
第五步：找到2节点，8都比2大，交换2、8位置
                  2
           |--------------|
           8              7
      |---------|    |---------|
      4         3    5         6
   |---
   1   
第6步：回到2节点的位置，2比3、4都小，交换2、最大值的位置
                  8
           |--------------|
           2              7
      |---------|    |---------|
      4         3    5         6
   |---
   1   
   
第7步：所有非终端节点都已经满足大根堆条件
                  8
           |--------------|
           4              7
      |---------|    |---------|
      2         3    5         6
   |---
   1

阶段式模型的结构类似于 CMM,它关注组织的成熟度。CMMI有5个成熟度等级:
    初始的:过程不可预测且缺乏控制。

    CLO(未完成的):过程域未执行或未得到CL1中定义的所有目标。

    CL3(己定义级的):其共性目标集中于已定义的过程的制度化。过程是按照组织的剪裁指南从组织的标准过程集中剪裁得到的，还必须收集过程资产和过程的度量，并用于将来对过程的改进。
    CL4(定量管理的):其共性目标集中于可定量管理的过程的制度化。使用测量和质量保证来控制和改进过程域,建立和使用关于质量和过程执行的定量目标作为管理准则。
    CL5(优化的):使用量化(统计学)手段改变和优化过程域，以满足客户要求的改变和持续改进计划中的过程域的功效。 
```

阶段式模型的结构类似于 CMM,它关注组织的成熟度。CMMI有5个成熟度等级:
    初始的:过程不可预测且缺乏控制。
    已管理的:过程为项目服务。
    已定义的:过程为组织服务。
    定量管理的:过程已度量和控制。
    优化的:集中于过程改进。
2.连续式模型
连续式模型关注每个过程域的能力，一个组织对不同的过程域可以达到不同的过程域能力等级(Capability Leve, CL)。CMMI中包括6个过程域能力等级，能力等级可以独立地应用于任何单独的过程域，任何一个能力等级都必须满足比它等级低的能力等级的所有准则。对各能力等级的含义简述如下:
    CLO(未完成的):过程域未执行或未得到CL1中定义的所有目标。
    CL1(已执行的):其共性目标是过程将可标识的输入工作产品转换成可标识的输出工作产品，以买现支持过程域的特定目标。
    CL2(已管理的):其共性目标集中于已管理的过程的制度化。根据组织级政策规定过程的运作将使用哪个过程，项目遵循文档化的计划和过程描述，所有正在工作的人都有权使用足够的资源，所有工作任务和工作产品都被监控、控制和评审。
    CL3(己定义级的):其共性目标集中于已定义的过程的制度化。过程是按照组织的剪裁指南从组织的标准过程集中剪裁得到的，还必须收集过程资产和过程的度量，并用于将来对过程的改进。
    CL4(定量管理的):其共性目标集中于可定量管理的过程的制度化。使用测量和质量保证来控制和改进过程域,建立和使用关于质量和过程执行的定量目标作为管理准则。
    CL5(优化的):使用量化(统计学)手段改变和优化过程域，以满足客户要求的改变和持续改进计划中的过程域的功效。 
```

    一个模块的各成分中间毫无关系。比如：一组语句在程序中的多处出现，为了节省内存空间，这些语句放在一个模块中，该模块的内聚是偶然内聚的。

2. 逻辑内聚：
    把几种逻辑上相关的功能放在同一个模块中。

3. 瞬时内聚（时间内聚）：
    一个模块所包含的任务必须在同一时间间隔内执行。例如初始化模块。

4. 过程内聚：
    一个模块的处理元素是相关的，而且必须按特定的次序执行。

5. 通信内聚：
    一个模块的所有成分都结合在同一个数据结构上。

6. 顺序内聚：
    模块的成分同一个功能密切相关且必须顺序执行，且输出，作为另一个成分的输入。（答案）

7. 功能内聚：
    模块内的所有成分属于一个整体，完成单一的功能。
```

在TCP/IP 网络中，建立连接进行可靠通信是在_传输层_完成的，此功能在OSI/RM七层模型中是在_会话层_来实现的。

在TCP/IP 网络中，将网络分为4层。功能如下:
(1)网络接口层
    TCP/IP参考模型没有对该层进行详细的描述，只是指出网络层可以使用某种协议与网络连接，负责实现IP数据报在不同通信系统中传输，实现物理地址（MAC地址)与逻辑地址(IP地址)相互映射的机制。
(2)网际层
    是TCP/IP参考模型的核心，负责IP数据报的产生以及IP数据报在逻辑网络上的路由转发，提供数据报的封装、分片和重组，以及路由选择和拥塞控制机制。
(3)传输层
    是TCP/IP参考模型提供端到端通信服务的层次，既可以提供面向连接的可靠通信服务，又可以提供无连接不可靠服务。该层以端口的形式实现通信复用。
(4)应用层
    是TCP/IP参考模型协议数量最多最复杂的层次,面向不同主题向用户提供各种各样的通信服务。66题选择D。

OSI/RM七层模型
主机：
    应用层：
        访问网络服务的接口；
        例如：为操作系统或网络应用程序提供访问网络服务的接口；
        常见：Telent、FTP、HTTP、SNMP、DNS等；
    表示层：
        提供数据格式转换服务；
        例如：解密与加密，图片解码和编码、数据的压缩和解压缩；
        常见：URL加密、口令加密、图片编解码；
    会话层：
        建立端连接并提供访问验证和会话管理（SESSION）；
        例如：使用校验点可使会话在通信失效时从校验点恢复通信；
        常见：服务器验证用户登录、断点续传；
    传输层：
        提供应用进程之间的逻辑通信；
        数据：数据段（Segment）；
        例如：建立连接，处理数据包错误、数据包次序；
        常见：TCP、UDP、SPX、进程、端口（socket）；
网络：
    网络层：
        为数据在结点之间传输创建逻辑链路，并分组转发数据；
        数据：分组（数据包）（Packet）；
        例如：对子网间的数据包进行路由选择；
        常见：路由器、多层交换机、防火墙、IP、IPX、RIP、OSPF；
    链路层：
        在通信的实体间建立数据链路连接；
        数据：帧（Frame）；
        例如：将数据分帧，并处理流控制、物理地址寻址、重发等；
        常见：网卡、网桥、二层交换机等；
    物理层：
        为数据端设备提供原始比特流的传输的通路；
        数据：比特（bit）；
        例如：网络通信的数据传输介质，由电缆与设备共同构成；
        常见：中继器、集线器、网线、HUB、RJ-45标准等；

在OSI/RM中，分为7层，理论上完成该功能是在会话层，但在定义时会话层实际完成功能较抽象,完成建立连接可靠同通信的是传输层中。
会话层：为彼此合作的表示层实体建立、维护和结束他们之间的通信会话，可以对同时双向的信息传输提供管理,也可以对任一时刻只能单向传输提供会话管理。

本题考查计算机网络中各层次知识点。传输层为会话层实体提供透明、可靠的数据传输服务，保证端到端的数据完整性，实现端到端的应答、分组排序和流量控制等功能。运输层有复用和分用功能，复用就是多个上层进程可同时使用下面传输层提供服务，分用则是运输层把收到的信息分别交付给上层中相应的进程。传输层信息的传送单位是报文。67题选择B。

### 2. (3)

```
程序查询方式：
    最简单的方式，也是数据传输速度最低的方式，当输入/输出控制器和外设交换数据时，CPU定时查询数据传输状态，发现外设传输完毕，再开始从输入/输出控制器中获取数据进行操作和处理，传输过程CPU全程参与。

中断方式：
    在I/O系统中，当CPU执行到I/O请求时，向输入输出控制器发出相应的指令后，CPU并不等待，而是继续执行其他操作，当数据完毕后，输入输出控制器向CPU发出中断请求，CPU将正在进行指令保护现场后，响应中断，并进行相应的处理,实现外设与CPU之间的并行工作。

DMA(Direct Memory Access)方式：
    指内存和I/O设备间传送一个数据块的过程，在DMA控制器的控制下，存储器和外设之间直接进行数据传送，CPU仅在开始时提供要传送的数据的起始地址和数据长度，在传送过程中不再参与，传送结束后，向CPU返回DMA操作完成中断请求信号，CPU将正在进行指令保护现场后响应中断，进行后续处理是一种快速传递大量数据常用的技术。实现外设与CPU之间的并行工作。
```

流密码(Stream Cipher)：
    也称为序列密码,它是对称密码算法的一种，利用密钥产生一个密钥流Z=Z1Z2Z3Z4...，然后利用此密钥流依次对明文X=X1X2X3X4...进行加密，这样产生的密码就是序列密码，也称流密码。密钥流由密钥流发生器f产生: Zi=f(k, Si)，这里的Si是加密器中存储器(记忆元件)在i时刻的状态，k是密钥。序列密码方案的发展是模仿“一次一密”系统的尝试。

序列密码与分组密码的对比：
    分组密码以一定大小作为每次处理的基本单元，而序列密码则是以一个元素(一个字母或一个比特)作为基本的处理单元。
    序列密码是一个随时间变化的加密变换，具有转换速度快、低错误传播的优点，硬件实现电路更简单;其缺点是:低扩散(意味着混乱不够)、插入及修改的不敏感性。
    分组密码使用的是一个不随时间变化的固定变换，具有扩散性好、插入敏感等优点;其缺点是:加解密处理速度慢、存在错误传播。

目前，公开的序列密码算法主要有RC4、SEAL等。AES是一种对称的分组加密算法。
```

数据结构循环队列知识点：
若用有N个元素的数组表示队列，随着一系列进队和出队运算，队列的结点移向存放队列的数组的尾端，会出现数组的前端空着，而队列空间已用完的情况(“假上溢”)。
一种可行的解决办法是当发生这样的情况时，把队列中的结点移到数组的前端，修改头指针和尾指针。
另一种更好的解决办法是采用循环队列。
循环队列就是将实现队列的数组a的第一个元素a[0]与最后一个元素 a[n- 1]连接起来。队空的初态为head=tail=0。在循环队列中，当tail赶上head时，队列满。反之，当head赶上tail时，队列变为空。这样队空和队满的条件都同为head=tail，这会给程序判别队空或队满带来不便。因此，可采用当队列只剩下一个空闲结点的空间时，就认为队列已满，用这种简单办法来区别队空和队满。使用了循环队列就避免了类似普通的顺序队列那种把队列中的结点移到数组的前端，修改头指针和尾指针以避免发生“假上溢”的问题，与此最接近的答案就只有B选项了。
```
